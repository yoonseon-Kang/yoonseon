# 전국통합식품영양성분 CSV 파일 가이드

## 파일 위치

CSV 파일 3개를 `public` 폴더에 다음 이름으로 배치해주세요:

```
public/
  ├── nutrition-data-1.csv
  ├── nutrition-data-2.csv
  └── nutrition-data-3.csv
```

## CSV 파일 형식

CSV 파일은 다음 컬럼을 포함해야 합니다 (헤더 필수):

### 필수 컬럼

| 컬럼명 | 설명 | 예시 |
|--------|------|------|
| 식품명 | 음식 이름 | 백미밥 |
| 식품대분류명 | 음식 카테고리 | 곡류 |
| 1회제공량 | 1회 제공량(g) | 100 |
| 에너지(kcal) | 칼로리 | 130 |
| 단백질(g) | 단백질 함량 | 2.5 |
| 탄수화물(g) | 탄수화물 함량 | 28.0 |
| 당류(g) | 당류 함량 | 0.1 |
| 지방(g) | 지방 함량 | 0.3 |
| 포화지방산(g) | 포화지방 함량 | 0.1 |
| 트랜스지방산(g) | 트랜스지방 함량 | 0.0 |
| 콜레스테롤(mg) | 콜레스테롤 함량 | 0 |
| 나트륨(mg) | 나트륨 함량 | 2 |

### 대체 컬럼명 (영어)

영어 컬럼명도 지원합니다:

- `name` → 식품명
- `category` → 식품대분류명
- `servingSize` → 1회제공량
- `calories` → 에너지(kcal)
- `protein` → 단백질(g)
- `carbs` → 탄수화물(g)
- `sugar` → 당류(g)
- `fat` → 지방(g)
- `saturatedFat` → 포화지방산(g)
- `transFat` → 트랜스지방산(g)
- `cholesterol` → 콜레스테롤(mg)
- `sodium` → 나트륨(mg)

## CSV 파일 예시

```csv
식품명,식품대분류명,1회제공량,에너지(kcal),단백질(g),탄수화물(g),당류(g),지방(g),포화지방산(g),트랜스지방산(g),콜레스테롤(mg),나트륨(mg)
백미밥,곡류,100,130,2.5,28.0,0.1,0.3,0.1,0.0,0,2
닭가슴살,육류,100,165,31.0,0.0,0.0,3.6,1.0,0.0,85,74
사과,과일,100,52,0.3,13.8,10.4,0.2,0.0,0.0,0,1
```

## 사용 방법

1. CSV 파일을 위의 형식에 맞게 준비합니다
2. 파일명을 `nutrition-data-1.csv`, `nutrition-data-2.csv`, `nutrition-data-3.csv`로 변경합니다
3. `public` 폴더에 파일을 배치합니다
4. 앱을 재시작하면 자동으로 데이터가 로드됩니다

## 주의사항

- CSV 파일 인코딩은 **UTF-8**이어야 합니다
- 숫자 값은 소수점을 사용할 수 있습니다 (예: 2.5)
- 빈 값이나 0인 경우 0으로 처리됩니다
- 파일명이 정확히 일치해야 합니다
- 헤더 행은 반드시 포함되어야 합니다

## 문제 해결

### 데이터가 로드되지 않는 경우

1. 브라우저 콘솔(F12)에서 에러 메시지 확인
2. CSV 파일 경로가 올바른지 확인 (`public` 폴더)
3. CSV 파일 인코딩이 UTF-8인지 확인
4. 컬럼명이 정확한지 확인

### 검색 결과가 나오지 않는 경우

1. CSV 파일에 데이터가 있는지 확인
2. `식품명` 컬럼에 값이 비어있지 않은지 확인
3. 브라우저를 새로고침하여 재시도
